# Мои кастомнный Docker-образ для нашего оператора CNPG

В рамках использования оператора CloudNative-PG (CNPG) есть [определенные требования к кастомному образу](https://cloudnative-pg.io/documentation/1.22/container_images/). 

В GitHub есть [соответствующие исходники](https://github.com/cloudnative-pg/postgres-containers). 

На основе этих исходников сделал свой кастомный образ, который, с одной стороны, удовлетворяет требованиям CNPG, но, с другой стороны, удовлетворяет и моим пользовательским требованиям, а именно: 

- дополнительные расширения для PostgreSQL (pg_wait_sampling, pg_stat_kcache, pg_cron, pg_profile)
- дополнительные пакеты и инструменты
- дополнительные настройки среды и предпочтения (в т.ч. файл .psqlrc с удобными псевдонимами для работы в утилите psql)

Мой кастомный образ CNPG (для PostgreSQL 16.2) я [опубликовал на DockerHub](https://hub.docker.com/r/kodmandvl/cnpg/tags), его можно использовать не только для работы с оператором CNPG, но и просто для запуска контейнеров с PostgreSQL в различных целях.
